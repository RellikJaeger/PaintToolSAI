2012-08-13 『关于现在的状况』 SAI 开发者：小松 浩司
　　停止更新太长时间了，先向一直抱有期待的各位致歉。虽然这不只是辩解，但我从 2009 年末开始就逐渐陷入了低潮，2010 年更是到了更为严重的状态，完全写不出半点程序。之后在 2011 年末左右，开始逐渐走出低潮，目前大约能够恢复以往 80% 左右的动力。

　　回到正题，关于 SAI 的开发状况，约从去年末开始，我就着手将 Ver.1 版的全部源代码删除，进而改写成新的源代码。我已经相当清楚会被说“你这家伙又…”，但是对于这份在持续的摸索和增建中变得杂乱的源代码，我的判断是与其增加新功能，不如用目前的技术重新设计，才能更早地完工。事到如今，虽然不好说敬请期待，但我已决定不管怎么样都要努力到完工为止，所以希望能得到您那么一点点的关注。

　　此外，以下是今后的粗略计划。新版将延续 Ver.1 的操作体系，只在提升性能的方向上进行开发。
　　•版本是 2 系列，但提供免费升级
　　•提供 32bit 版和 64bit 版
　　•完全支持多核心
　　•增加所有在 Ver.1 版预定的新功能

2012-12-03 Ver.1.2.0-Beta.4
*** 已验证了本软件能在 Windows 8 上运行，但结论上是不推荐使用 Windows 8 ***
在自 Windows Vista 开始引入的 Aero 桌面模式上，程序的绘画结果反映到屏幕上会有20毫秒左右的延迟。
虽然这个数字看似很轻微，但如果是在玩对时机很严格的动作游戏上，这个时间可是漫长到足以产生不协调感。
即使是 SAI，也会对小直径笔迹的直接感有很大的损失。即使扣除开发者自身的偏袒，也能感受得出。

幸运的是到 Windows 7为止，终端用户可自行手动将桌面更改为 Basic 模式和经典模式来解决这个延迟。
可是，Windows 8 取消了 Basic 模式和经典模式，导致这个延迟变得不可回避。
其结果，作为 SAI 的最大卖点的小直径笔迹的直接感会大为损失，这就是我不推荐使用 Windows 8 的理由。
尽管如此，如果您原本就在 Aero 下使用 SAI，并对这个程度的延迟不太在意的话，我觉得您在 Windows 8 中使用 SAI 也不会有什么问题。

•修正在 Windows 8 的多屏显示模式下，在副显示屏上窗口最大化时，会隐藏任务栏的问题
•数位板的屏幕映射范围改为从平板电脑的驱动程序中获取(取消了对 WACOM 旧版驱动的 BUG 对策)
•修正在编辑框用 IME 输入法时，用回车键之外的按键确定输入内容时，正在转换的的文字的显示位置不会移动的问题
•修正不能用拖放方式打开扩展名为“.jpeg”等文件的问题
•修正在文件浏览器中双击文件夹的快捷方式，无法打开链接目标的问题
•修正不能保存到 Windows XP 网络邻居的根目录的问题
•其他的细微调整
